# 🐾 Argus — AI-Powered Code Review for Every PR

> Every pull request. Every team member. The same standard.

Argus is a multi-agent AI pipeline that automatically reviews pull requests for security vulnerabilities, performance issues, code quality, and missing tests — and posts a detailed report as a PR comment.

No IDE plugins. No manual triggers. Just open a PR.

---

## How It Works

When a developer opens a pull request, Argus automatically:

1. Reads the PR diff
2. Runs 4 specialized AI agents in parallel
3. Resolves conflicts between agents via a Debate Agent
4. Posts a structured markdown report as a PR comment
5. *(Optional)* Opens an auto-fix branch for critical issues

```
PR opened
    ↓
Security Agent ──┐
Performance Agent ─┼──→ Debate Agent ──→ PR Comment
Style Agent ──────┤
Test Agent ───────┘
```

---

## Agents

| Agent | What it does |
|-------|-------------|
| 🔒 **Security** | Scans for OWASP Top 10: hardcoded credentials, SQL injection, XSS, unsafe imports |
| ⚡ **Performance** | Detects O(n²) loops, N+1 database queries, repeated expensive computations |
| ✨ **Style** | Flags naming violations, dead code, missing docstrings, high complexity |
| 🧪 **Test** | Identifies untested functions and missing edge case coverage |
| ⚖️ **Debate** | Resolves conflicts when agents disagree on severity, produces final report |

## Installation

### Step 1 — Create the workflow file

In your repository, create `.github/workflows/review.yml`:

```yaml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  pull-requests: write
  issues: write
  contents: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: BakemonoTR/Argus@v1.9
        with:
          groq_api_key: ${{ secrets.GROQ_API_KEY }}
```

### Step 2 — Get a Groq API key

1. Go to [console.groq.com](https://console.groq.com)
2. Create a free account
3. Generate an API key

### Step 3 — Add the secret to your repository

1. Go to your repository on GitHub
2. **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Name: `GROQ_API_KEY`, Value: your Groq API key

### Step 4 — Done

From now on, every pull request will automatically receive an AI code review.

## Options

### Language support

Argus can write summaries in different languages:

```yaml
- uses: BakemonoTR/Argus@v1.9
  with:
    groq_api_key: ${{ secrets.GROQ_API_KEY }}
    language: 'tr'   # tr, es, de, fr, jp...
```

### Auto-fix branch

When critical issues are found, Argus can automatically open a fix branch:

```yaml
- uses: BakemonoTR/Argus@v1.9
  with:
    groq_api_key: ${{ secrets.GROQ_API_KEY }}
    autofix: 'true'
```

This creates a `autofix/pr-{number}` branch with AI-suggested fixes applied. **Review the fixes before merging** — treat them as suggestions, not guaranteed corrections.

## Severity Scale

| Badge | Score | Meaning |
|-------|-------|---------|
| 🔴 | 8–10 | Critical — must be addressed before merge |
| 🟡 | 5–7  | Warning — should be reviewed |
| 🟢 | 1–4  | Info — minor improvement suggestion |

Issues with severity ≥ 8 are listed as **Critical Issues** and trigger the auto-fix branch (if enabled).

## Why Argus?

Most AI coding tools (Copilot, Cursor, etc.) help individual developers while they write code. Argus works at the team level — enforcing the same quality standard on every PR, regardless of which IDE each developer uses.

| | IDE AI Assistants | Argus |
|---|---|---|
| Works per developer | ✅ | ✅ |
| Works per team | ❌ | ✅ |
| Enforced on every PR | ❌ | ✅ |
| Leaves an audit trail | ❌ | ✅ |
| Requires setup per machine | ✅ | ❌ |

## Limitations

- **LLM accuracy** — Argus catches obvious issues reliably. Complex business logic or framework-specific patterns may be missed.
- **Auto-fix quality** — AI-generated fixes should always be reviewed by a human before merging.
- **Groq rate limits** — Free tier allows ~30 requests/minute. On very large PRs, consider reducing the number of active agents.
- **Python only** — Current agents are optimized for Python code. Other languages will still be analyzed but with reduced accuracy.
